🌳 The log n Proof, In One Sentence

👑

Always Keep the Biggest

At every node, identify the child with the largest subtree — the "heavy" child. Its data structure is never rebuilt; it's simply reused as the parent's own structure.

🔀

Only Small Ones Get Merged

Every OTHER child ("light" children) gets its elements inserted one by one into the kept structure, then discarded. This is the only work actually done at each node.

📐

Why log n Moves Is the Ceiling

Every time an element gets merged as part of a "light" subtree, the subtree containing it at least DOUBLES in size — from the light subtree's size to something bigger than the heavy sibling it merged into. Size can only double log n times before reaching n.

🏆

Same Trick as Union-Find

This is the exact same "small-to-large" idea behind union-by-size in Disjoint Set Union — always attach the smaller structure to the larger one. DSU on Tree just applies it to per-node subtree processing order instead of set merging.
