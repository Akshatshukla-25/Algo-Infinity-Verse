Succinct Bit Vector Explorer

n + o(n) bits. O(1) rank and select. No decompression step.

A succinct bit vector answers "how many 1s before position i?" (rank) and "where is the k-th 1?" (select) in constant time, using almost no extra space beyond the raw bits themselves. This exact structure is the backbone of the FM-index — the data structure that lets tools like BWA and Bowtie search a human genome for a DNA substring in milliseconds.

🧬 How O(1) Rank Actually Works

📦

Two-Level Decomposition

Split n bits into superblocks (e.g. 8 bits), each split into blocks (e.g. 4 bits). Precompute the cumulative rank at the start of every superblock and every block. This costs O(n / log n) extra bits total — sublinear, hence "succinct."

⚡

The Popcount Table

Within a block, don't count bits one by one — look up the popcount of the entire block pattern in a precomputed table (only 16 entries for 4-bit blocks). This lookup is O(1), completing the rank query in constant time overall.

🔍

Select via Binary Search

select(k) finds the position of the k-th 1 bit. Binary search over superblock ranks locates the right superblock in O(log(n/superblock_size)), then a linear or binary scan within the block finds the exact bit — still effectively O(1) for fixed block sizes.

🧬

FM-Index for Genomics

The FM-index applies rank/select to a Burrows-Wheeler transformed genome. Searching for a DNA substring of length m takes O(m) rank queries — regardless of genome size — which is why aligning short reads to a 3-billion-base human genome takes milliseconds, not hours.
