What is happening?

Without the outbox pattern, an app might commit to the DB and then crash before publishing to Kafka (or vice versa), leading to data inconsistency. By writing the Order and the Event to an Outbox table in a single local transaction, we guarantee atomicity. A separate relay process then reads the Outbox and publishes to Kafka.
